**4. Core Concepts**

**Concept 1: Chunked Transfer Encoding**

- `Transfer-Encoding: chunked` = server gửi HTML từng phần
- Nếu thấy `Content-Length: 12345` = buffer đang bật, streaming chết

**Concept 2: Các layer buffer**

| Layer      | Buffer khi nào                | Fix                           |
| ---------- | ----------------------------- | ----------------------------- |
| Nginx      | `proxy_buffering on` mặc định | `proxy_buffering off`         |
| Cloudflare | Auto Minify HTML              | Tắt Auto Minify               |
| AWS Lambda | Mặc định không stream         | `InvokeMode: RESPONSE_STREAM` |
| Gzip       | Nén cần đủ data               | `compress: false`             |

**Concept 3: PPR + Streaming**

- Shell prerender tại build → cache ở edge
- Hole stream tại request → TTFB ~0ms
- Nếu buffer → shell + hole gửi cùng lúc, mất lợi PPR

**Concept 4: Standalone Output**

- `output: 'standalone'` tạo minimal image
- Phải tự copy `public/` và `.next/static/`

**5. Legacy Recognition**

Dấu hiệu nhận biết

✅

- `getServerSideProps` đợi tất cả data → TTFB chậm
- Không có `Suspense` → không có streaming
- `next export` → static HTML, không stream

❌

- `Suspense` nhưng proxy buffer → "fake streaming"
- PPR bật nhưng CDN cache HTML động → stale shell

| Legacy                               | Modern                                |
| ------------------------------------ | ------------------------------------- |
| `getServerSideProps` + đợi full data | `Suspense` + stream từng phần         |
| `next export` static                 | PPR: shell static + hole dynamic      |
| Nginx mặc định buffer                | `proxy_buffering off`                 |
| Lambda mặc định                      | Lambda Function URL `RESPONSE_STREAM` |
| Copy toàn bộ node_modules            | `output: 'standalone'`                |

**7. Production Scenario**

**Context:** E-commerce product page, deploy VPS + Nginx + Cloudflare

**Problem:** User báo trang load chậm, skeleton không hiện

**Root Cause:** Nginx `proxy_buffering on`, Cloudflare Auto Minify bật

**Solution:**

```nginx
location / {
    proxy_pass http://nextjs:3000;
    proxy_buffering off;
    proxy_cache off;
}
```

```
Cloudflare Page Rule:
  Auto Minify HTML: OFF
  Cache Level: Bypass (cho /product/*)
```

**Lesson:** Streaming chỉ có ý nghĩa khi **mọi layer** forward chunk ngay.

👉 Rule:

> Nếu gặp **TTFB chậm dù đã dùng Suspense** hãy nghĩ ngay đến **proxy buffer**.

## 1. Lesson Goal

Hiểu tại sao streaming của Next.js App Router **không tự nhiên chạy được** ở mọi hạ tầng. Biết cách cấu hình từng loại hạ tầng (reverse proxy, CDN, serverless, container) để tận dụng trọn vẹn Suspense, PPR, và TTFB tối ưu. Đồng thời nhận ra trade-offs giữa "zero-config" (Vercel) và "full-control" (self-host).

## 2. Evolution Context

| Era                               | Cách deploy                     | Vấn đề streaming                                                                                                |
| --------------------------------- | ------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| Next.js 9–12 (Pages Router)       | `next start` hoặc `next export` | Không có streaming. SSR trả full HTML một lần.                                                                  |
| Next.js 13–14 (App Router)        | Vercel hoặc self-host           | Streaming có nhưng proxy/CDN hay buffer.                                                                        |
| Next.js 15+                       | `htmlLimitedBots` config        | Bot detection tùy chỉnh.                                                                                        |
| Next.js 16 (PPR stable, Oct 2025) | `ppr: true`, `cacheComponents`  | Shell prerender + hole stream. Theo Next.js blog chính thức, PPR stable release là Next.js 16 vào October 2025. |

## 3. Mental Model

**Streaming giống như giao đồ ăn từng món:**

- Bếp (Next.js) nấu xong món nào giao món đó.
- Nhưng nếu người giao hàng (proxy/CDN) đợi đủ cả bữa mới giao → khách đói.
- **Mỗi layer giữa server và client có thể "đợi đủ rồi mới giao" → streaming bị phá hỏng.**

Theo Next.js docs chính thức, "Any layer between your server and the client that buffers the response can diminish the benefits of streaming."

## 4. Core Concepts

### 4.1 HTTP Chunked Transfer Encoding

Streaming dựa trên `Transfer-Encoding: chunked`. Server gửi HTML từng phần, không cần biết tổng kích thước trước.

```bash
# Kết quả mong muốn
curl -i -N http://localhost:3000/product/1

HTTP/1.1 200 OK
Transfer-Encoding: chunked
Content-Type: text/html; charset=utf-8

<html>...          ← chunk 1 (shell)
<script>...        ← chunk 2 (metadata)
<div id="S:1">...  ← chunk 3 (hole data)
```

Nếu thấy `Content-Length: 12345` thay vì `Transfer-Encoding: chunked`, streaming đang bị chặn.

### 4.2 Các layer có thể buffer

| Layer                        | Cách buffer                   | Cách phát hiện                 |
| ---------------------------- | ----------------------------- | ------------------------------ |
| Reverse proxy (Nginx)        | `proxy_buffering on` mặc định | `curl -I` không thấy `chunked` |
| CDN (Cloudflare)             | Auto Minify HTML              | TTFB ≈ Total time              |
| Load balancer (ALB)          | WAF/Lambda@Edge transform     | Response delay đều đặn         |
| Compression (Gzip)           | Nén cần đủ data               | Chunk đầu delay bất thường     |
| Serverless (Lambda mặc định) | Trả toàn bộ response một lần  | Không có `chunked`             |

Theo Next.js docs self-hosting, "Nginx and similar reverse proxies buffer responses by default."

## 5. Legacy Recognition

**Anti-pattern từ Pages Router era:**

```js
// pages/product/[id].js — Pages Router, không có streaming
export async function getServerSideProps() {
  const data = await fetchSlowAPI(); // Đợi 3 giây
  return { props: { data } };
}

// User đợi 3 giây nhìn trang trắng
// Không có Suspense, không có fallback
```

**App Router đúng:**

```tsx
// app/product/[id]/page.tsx — Streaming
export default function ProductPage() {
  return (
    <main>
      <h1>Product</h1> {/* Shell: render ngay */}
      <Suspense fallback={<Skeleton />}>
        <SlowData /> {/* Hole: stream sau */}
      </Suspense>
    </main>
  );
}
```

**Nhưng** nếu proxy buffer, App Router cũng trở thành "getServerSideProps 2.0" — user vẫn đợi 3 giây.

## 6. Modern Approach

### 6.1 Reverse Proxy — Nginx

**Vấn đề:** Nginx buffer response by default.

**Giải pháp 1:** Header từ Next.js:

```js
// next.config.js
module.exports = {
  async headers() {
    return [
      {
        source: '/:path*{/}?',
        headers: [{ key: 'X-Accel-Buffering', value: 'no' }],
      },
    ];
  },
};
```

**Giải pháp 2:** Cấu hình Nginx trực tiếp:

```nginx
location / {
    proxy_pass http://nextjs:3000;
    proxy_http_version 1.1;
    proxy_buffering off;   # ← TẮT buffering
    proxy_cache off;       # ← Không cache HTML động
    proxy_read_timeout 60s;
}
```

**Kiểm chứng:**

```bash
curl -i -N http://your-site.com/product/1 | grep "Transfer-Encoding"
# Kết quả đúng: Transfer-Encoding: chunked
```

### 6.2 CDN — Cloudflare

**Vấn đề:** "Auto Minify HTML" phải đọc toàn bộ HTML để minify → buffer.

**Giải pháp:**

```
Page Rules:
  URL: *example.com/product/*
  Settings:
    Cache Level: Bypass        ← Không cache HTML động
    Auto Minify HTML: OFF       ← Tắt minify
```

### 6.3 Serverless — AWS Lambda

**Vấn đề:** Lambda mặc định trả toàn bộ response một lần. Không stream.

**Giải pháp:** Lambda Function URL với `InvokeMode: RESPONSE_STREAM`.

```yaml
# SAM template
Resources:
  NextJSFunction:
    Type: AWS::Lambda::Function
    FunctionUrlConfig:
      AuthType: NONE
      InvokeMode: RESPONSE_STREAM # ← BẮT BUỘC
```

**Hoặc:** Dùng ECS/Fargate thay vì Lambda nếu cần streaming đơn giản.

Theo Vercel blog (March 2023), "Vercel's serverless architecture is largely powered by AWS Lambda, which... does not provide native support for HTTP streaming." Vercel đã phát triển custom solution để enable streaming trên nền Lambda.

### 6.4 Docker — Standalone Output

Theo Next.js docs chính thức, `output: 'standalone'` tạo minimal image chỉ chứa file cần thiết runtime.

```dockerfile
# Multi-stage build
FROM node:20-alpine AS builder
WORKDIR /app
COPY . .
RUN npm ci && npm run build

FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production

# Standalone output chỉ cần 3 thư mục
COPY --from=builder /app/public ./public
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static

EXPOSE 3000
ENV PORT=3000
ENV HOSTNAME="0.0.0.0"
CMD ["node", "server.js"]
```

**Lưu ý:** Standalone mode không tự động copy `public` và `.next/static`. Phải copy thủ công.

## 7. Production Scenario

### Scenario 1: E-commerce với PPR + Self-host

**Yêu cầu:**

- Product page: shell tĩnh (prerender), giá và reviews động (stream)
- Deploy trên VPS + Nginx
- CDN Cloudflare cho static asset

**Kiến trúc:**

```
User → Cloudflare (static cache) → Nginx (proxy_buffering off) → Next.js (PPR)
         ↓
    /_next/static → cache 1 năm
    /product/*    → bypass cache (HTML động)
```

**Config:**

```js
// next.config.js
module.exports = {
  output: 'standalone',
  experimental: {
    ppr: true, // Next.js 16+
  },
  async headers() {
    return [
      {
        source: '/:path*{/}?',
        headers: [{ key: 'X-Accel-Buffering', value: 'no' }],
      },
    ];
  },
};
```

```nginx
# nginx.conf
server {
    listen 443 ssl http2;

    location / {
        proxy_pass http://nextjs:3000;
        proxy_buffering off;
        proxy_cache off;
    }

    location /_next/static {
        proxy_pass http://nextjs:3000;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

### Scenario 2: SaaS Dashboard trên Vercel

**Yêu cầu:**

- Dashboard personalized theo user
- Không thể cache HTML (vì personalized)
- Vẫn muốn TTFB tốt

**Giải pháp:**

```tsx
// app/dashboard/page.tsx
export const dynamic = 'force-dynamic'; // Không cache HTML
export const runtime = 'nodejs'; // Node.js runtime cho streaming

export default function Dashboard() {
  return (
    <main>
      <DashboardHeader /> {/* Shell: nhanh */}
      <Suspense fallback={<StatsSkeleton />}>
        <UserStats /> {/* Hole: stream */}
      </Suspense>
    </main>
  );
}
```

**Trade-off:** Không dùng PPR (vì không cache được shell personalized). Streaming thuần vẫn giúp TTFB.

## 8. Debugging Scenario

### Problem: Streaming không hoạt động sau deploy

**Symptom:** User báo trang load chậm, skeleton không hiện.

**Debug steps:**

```bash
# Step 1: Kiểm tra chunked encoding
curl -i -N https://site.com/product/1 | head -5

# Mong đợi: Transfer-Encoding: chunked
# Thực tế: Content-Length: 15420  ← Buffer đang bật!

# Step 2: Kiểm tra TTFB
curl -o /dev/null -s -w "TTFB: %{time_starttransfer}s\n" \
  https://site.com/product/1

# TTFB ≈ Total time → streaming bị chặn

# Step 3: Kiểm tra từng layer
# Test trực tiếp Next.js (bypass Nginx)
curl -i -N http://localhost:3000/product/1 | grep "Transfer-Encoding"
# Nếu có chunked → lỗi ở Nginx
# Nếu không có → lỗi ở Next.js config
```

**Root cause:** Nginx `proxy_buffering on` mặc định.

**Fix:**

```nginx
location / {
    proxy_buffering off;
    proxy_cache off;
}
```

## 9. Refactoring Scenario

### Từ: SSR thuần (buffer toàn bộ)

```tsx
// Trước: Không có Suspense, đợi tất cả data
export default async function Page() {
  const product = await fetchProduct(); // 1 giây
  const reviews = await fetchReviews(); // 3 giây
  const related = await fetchRelated(); // 2 giây

  return (
    <main>
      <h1>{product.name}</h1>
      <Reviews data={reviews} />
      <RelatedProducts data={related} />
    </main>
  );
}
// TTFB = 6 giây, user nhìn trang trắng
```

### Sang: Streaming với Suspense

```tsx
// Sau: Shell render ngay, hole stream dần
export default function Page() {
  return (
    <main>
      <h1>Product</h1> {/* Shell: ngay lập tức */}
      <Suspense fallback={<ProductSkeleton />}>
        <ProductInfo /> {/* Stream sau 1 giây */}
      </Suspense>
      <Suspense fallback={<ReviewSkeleton />}>
        <ReviewList /> {/* Stream sau 3 giây */}
      </Suspense>
      <Suspense fallback={<RelatedSkeleton />}>
        <RelatedProducts /> {/* Stream sau 2 giây */}
      </Suspense>
    </main>
  );
}
// TTFB = 50ms, user thấy layout ngay
```

**Nhưng** nếu Nginx buffer, cả hai đoạn code trên đều cho TTFB = 6 giây. **Streaming chỉ có ý nghĩa khi hạ tầng hỗ trợ.**

## 10. Architecture Notes

### 10.1 Vercel vs Self-hosting Trade-offs

Theo Divotion blog (July 2025), "Next.js works best when you run it on Vercel. Many features are built with Vercel in mind, and some of them only work there."

|                | Vercel              | Self-host (Docker + Nginx)      |
| -------------- | ------------------- | ------------------------------- |
| Streaming      | Native, zero-config | Cần cấu hình proxy              |
| PPR            | Native              | Cần Next.js 16+, cấu hình cache |
| Preview deploy | Built-in            | Cần CI/CD tự xây                |
| Cost           | $20+/user/tháng     | $5-20/server/tháng              |
| Control        | Limited             | Full                            |
| Cold start     | Edge: ~25ms         | Container: ~1-3 giây            |

### 10.2 Multi-instance Cache Coordination

Theo Next.js docs self-hosting, "App Router deployments with multiple instances need cache tag coordination."

```js
// Custom cache handler với Redis
// next.config.js
module.exports = {
  cacheHandler: require.resolve('./cache-handler.js'),
};
```

```js
// cache-handler.js
const { CacheHandler } = require('@neshca/cache-handler');
const createRedisHandler = require('@neshca/cache-handler/redis-stack');

module.exports = class extends CacheHandler {
  constructor(options) {
    super(options);
    this.handlers = [createRedisHandler({ client: redisClient })];
  }

  // refreshTags() được gọi trước mỗi request
  async refreshTags() {
    // Sync tag state từ Redis để tất cả instances biết invalidation
  }
};
```

### 10.3 Build ID cho PPR Cross-deploy

```js
// next.config.js
module.exports = {
  generateBuildId: () => process.env.GIT_SHA,
};
```

Tránh lỗi hydration mismatch khi shell cũ trong cache không khớp runtime mới.

**Q3: Tại sao PPR cần streaming support ở hạ tầng?**

<details>
<summary>Đáp án</summary>

PPR gửi shell prerender trước, sau đó stream hole vào. Nếu hạ tầng buffer, shell và hole được gửi cùng lúc sau khi tất cả resolve xong. Theo Next.js docs, "Without it, the static shell and dynamic content are delivered together after the full render completes, eliminating PPR's time-to-first-byte advantage."

</details>

**Q4: Khi nào nên dùng Vercel thay vì self-host?**

<details>
<summary>Đáp án</summary>

Vercel khi: team không có DevOps, cần preview deploy, traffic biến động, muốn zero-config. Self-host khi: cần full control, cost-sensitive, có team DevOps, hoặc cần feature không support trên Vercel (WebSocket, long-running process). Theo Divotion, "The line between Next.js and Vercel is paper thin, and that is a problem if you ever want to leave."

</details>

## 12. Exercises

**Exercise 1:** Viết Dockerfile cho Next.js App Router với standalone output, đảm bảo copy đúng `public` và `.next/static`.

<details>
<summary>Đáp án</summary>

```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production

# Standalone không tự copy public và static
COPY --from=builder /app/public ./public
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static

EXPOSE 3000
ENV PORT=3000
ENV HOSTNAME="0.0.0.0"
CMD ["node", "server.js"]
```

Lưu ý: `public` chứa favicon, robots.txt. `.next/static` chứa CSS/JS build. Thiếu một trong hai sẽ 404.

</details>

**Exercise 2:** Cấu hình Nginx để hỗ trợ streaming cho Next.js App Router.

<details>
<summary>Đáp án</summary>

```nginx
server {
    listen 443 ssl http2;
    server_name app.example.com;

    location / {
        proxy_pass http://nextjs:3000;
        proxy_http_version 1.1;
        proxy_buffering off;      # ← Tắt buffering
        proxy_cache off;          # ← Không cache HTML động
        proxy_read_timeout 60s;
    }

    location /_next/static {
        proxy_pass http://nextjs:3000;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

Hoặc set header từ Next.js: `X-Accel-Buffering: no`.

</details>

## 13. Summary

- Streaming dựa trên HTTP Chunked Transfer Encoding. Một layer buffer là đủ phá hỏng.
- Nginx: `proxy_buffering off` hoặc `X-Accel-Buffering: no`.
- Cloudflare: Tắt Auto Minify HTML, bypass cache cho route động.
- AWS Lambda: Dùng Function URL với `RESPONSE_STREAM` hoặc chuyển ECS.
- Docker: `output: 'standalone'`, nhớ copy `public` và `.next/static`.
- PPR cần streaming support — nếu không, mất lợi ích TTFB.
- Vercel vs self-host: trade-off giữa zero-config và full control.
